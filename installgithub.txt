установка git

https://git-scm.com/downloads           
----------------------------------------------------------------------------------------

Инициализация git (создание локального репозитория)

git init

----------------------------------------------------------------------------------------

Базовые настройки (должны совпадать с данными на github)

git config --global user.name "OlegFD11" - имя пользователя
git config --global user.email logworkb@gmail.com  - мэил пользователя
git config --list (показывает файл конфигурации проекта в git)
git help "команда" (показывает справку по выбранной команде например git help init )

-----------------------------------------------------------------------------------------


Базовые операции

git status  (текущее состояние файлов с изменениями)

git add .     (добавляет изменненые файлы в локальный репозиторий git)

git commit  -m"add file"   (фиксирует слепок/снимок файлов на текущий момент)



-----------------------------------------------------------------------------------------

работа с ветками

git branch -v (показывает список веток)

git branch 

git checkout имяветки  (переключение на конкретную ветку)

git checkout -b branch3 (создание и переход в созданную ветку)

git merge имя сливаемой ветки (слияние веток)

---------------------------------------------------------------------------------------

возврат локального репозитория до состояния опредленного коммита

git log
q - выход из просмотра log (если не происходит выход из log)
git checkout -b Новая ветка индекс комита

---------------------------------------------------------------------------------------

Выгрузить файлы на github


git remote add origin https://github.com/OlegKozintsev/distfile.git
git push -u origin master


----------------------------------------------------------------------------------------

Подключиться к работе над проектом

git clone   - сделать локальную копию на компьютере

git pull    - выгрузить полученные файлы в папку с копией проекта

---------------------------------------------------------------------------------------

отправить изменненные файлы на github

git push   (синхронизирует/отправляет файлы на удаленный репозиторий)

---------------------------------------------------------------------------------------

получить актуальную версию проекта

git pull
